Template Help

Templates are a feature that most Email Service Providers (ESP's) support which gives a user the ability to save a common re-usable email structures.  Often, as is in this case, the template supports the ability to merge dynamic data into the template just before sending in order to personalize the email either at a group level (everyone getting the current email) or even down to the individual user. 

When using templates, common headers, footers, colors and general email structure can be saved off so each use of the template will keep your branding consistent.  Templating also helps speed the process of change.  Often, homegrown email systems have locked down email formats that can only be changed by development.  As we all know, development changes can often take a long time to work through the typical change process.  But when using a system that allows for the user to change the template via a user interface, changes can be made quickly by many different people outside of development.  Depending on the complexity of the change, development's help may be needed, but that tends to be a rare occurance.

This template manager is build on top of the SparkPost email platform and interfaces with the SparkPost server via their API's, never keeping data except within your own SparkPost account.  The SparkPost platform supports two versions of a template at any given time; Draft and Published.  This template manager also gives you access to both those versions.  

At the top of the page, the user has the option to either enter in the name of a new template they wish to create, or to select a stored template from the pulldown.  On coming to this page, the system will automatically go to the users SparkPost account and retrieve all templates available to that user.  Available templates may conprise of two groups of templates; the templates created by this user, and those created by the master account and shared with all users.  Those shared from the master account are read only and can not be changes.  They can be copied though by taking the following steps:

  1. Select a shared template from the Stored Template pull down
  2. Enter in a new template name at the top of the page
  3. Under the Draft template editing section, press either the Save Draft or Save and Publish button. 

Now you have an editable version of that shared template.  If the master copy is changed at a latter time, those changes will not affect your copy.

Draft

As discussed earlier, there can be two copies of your template, Draft and Published.  You should think of them as two seperate entities.  Also, each copy has both a HTML and Text version of the template.  Two move from the Draft or Published copies of your template, simply select the appropriate link under the Template Code/Text header.   When working with the Draft copy of the HTML template, you are presented with an embedded WYSIWYG HTML editor.  Since most editors are similar, this help section will not be a tutorial on using the editor.  Under the draft section you are also able to set up your Email Text section.  Since the email text section is simple text, you are only presented with a simple text dialog box for entering in that data.  If you choose to save off the draft, or even publish the draft, you may do so within either Draft page.  The Preview tab/link allows you to see what the email may look like.   If you have entered in data within the Test Recipient Data section, that data will be used to demonstrate what they email may look like.  If data for more than one recipient has been entered; only the data for the first recipient will be used for the Preview.

Please keep in mind, that when you use the Save and Publish button, you are writing over your current published version. 

Published

The Published section is similar in the Draft section, in the fact that it have both an HTML and Text section; but it's different, in the fact that it's READ ONLY.   Yes, Published template data can not be modified.  If you want to modify that code/html, you must copy the HTML/TEXT data over to the Draft section for editing.  There is a handy button in the Published section to help with that.  Keep in mind, the Copy button will overwrite your current draft data.  Also, the supporting data like From, Reply To, Subject, etc, are NOT copied from the Published template to Draft.  You must go to the bottom of the page and make those changes yourself.

* note * If you copy published code to the draft fields, that data will only be kept once you press the Save Draft or Save and Publish button.

You can also Preview your published code by going to the Preview button in the Published section.

Quicktest

This is where things get a little complicated.  At the bottom of the Template Manager page are buttons that allow you to send test emails to email address that have been entered into the Test Recipient Data section.  That powerful option gives you the ability to send a unique email to each user with their own data set.  That data must be entered in either in CSV or JSON format; which depending on your template, can be very complex.  The Help tab within that section will discribe how this is done in detail.

But sometimes you don't want to enter in data and you just want to see what the email will look like even if it might look poorly without data.  This is where Quicktest comes in.  Quicktest takes a list of comma seperated email addresses and will send the same email to all users entered into the Quicktest data field.  Even if there is data within the Test Recipient Data fields, they will NOT be used.  To use that data, you must use the buttons toward the bottom of the page.

Substitution Fields

SparkPost has a very rich substitution capability that can be used to personalize any email.  For the most part, substitution fields are represented within the HTML by double curly brackets.  For example, if you want to personalize the greeting, you might have the following HTML code in the template:

<h3>Hello {{first_name}},</h3>

This is a very simple example of embedding the substitution fields within the HTML code.  In reality, you would probably have a default entry just in case the first name wasn't available:

<h3> Hello {{ name or 'Customer' }}

SparkPost also has other conditional capabilities, the ability to loop through arrays of data and a myraid of other capabilities. For a full explanation of the substitution capabilities that are available to you, please refer to the documentation at SparkPost::
